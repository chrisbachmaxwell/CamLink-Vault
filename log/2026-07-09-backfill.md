# 2026-07-09 — Backfill: the whole build so far

One dated chronicle of the project from zero to two working cameras.
Detail lives in the linked pages; this is the narrative spine.

## Phase 1 — Platform (early July 2026)
Monorepo, core SDK (client, adapter interfaces, CaptureSession with retry
queue + manifests + SHA-256), MockCameraAdapter, 9-check certification
suite, CLI, CI, docs. Then the CCAPI adapter + simulator — until Chris's
R10 revealed no CCAPI menu without activation → pivot decision: build
"the Tether Tools solution" = [[ptp-ip-protocol]] (activation-free), with
USB tether planned next. Clinic app built so Chris could test as a
dr. office; wizard added because "I want our app to walk us through
connecting the camera."

## Phase 2 — The discovery war (R10, home network)
Symptom: camera stuck at "start EOS Utility on the computer"; app hears
nothing. In order, the layers we peeled (each now a page):
1. Camera searches via SSDP BEFORE connecting; nobody answered →
   built the [[eos-utility-pairing]] announcer (Windows SERVER header!).
2. Bare TCP reach-probes aborted the camera's pairing → graceful FIN,
   probe folded into pairing.
3. ECONNRESET crashed the whole app → socket error handlers + process guards.
4. SetRemoteMode → 0x2002 → [[remote-shutter-degraded-mode]].
5. tcpdump PROVED M-SEARCH reached the Mac; app heard nothing →
   `lsof` showed **Spotify + node both on UDP 1900** → SO_REUSEPORT +
   port-contention detection. Spotify quit; still nothing.
6. Announcer self-test built (loopback/multicast probes) → verdict:
   socket fine, network inbound dead. Firewall OFF (ruled out).
7. **macOS Local Network permission**: Terminal absent from the list,
   prompt never fired (macOS 26.5.1), ping "prohibited by filter" exposed
   BOTH a stuck TCC prompt and [[home-network-filter]] (router blocks
   peer traffic — Same-Wi-Fi mode parked).
8. **Reboot re-armed the permission popup → discovery worked.**
Also fixed en route: browser cached stale UI (→ no-store + visible
version), misleading scan banner, wizard decluttered to three tiles at
Chris's request.

## Phase 3 — Connection & capture (R10, camera's own Wi-Fi)
- All checks green, then ECONNRESET + camera AP restart → the
  check-then-redial bug → [[session-handoff]] (the connection that passes
  the checks becomes the working connection).
- macOS auto-hop mid-handshake → 2-min retry + "rejoin EOS-XXXXXX" banners
  + Auto-Join OFF instruction.
- Shutter press → nothing: my doc-derived ObjectAddedEx layout was wrong
  (libgphoto2 comparison) AND the test screen ignored non-session photos →
  layout-proof events (oid only + GetObjectInfo), unhandled-event tracer,
  test step reacts to physical shutter.
- **R10 delivered photos end-to-end. First pipeline proof.**

## Phase 4 — Second body in hours (R6 Mark III, 2026-07-09)
New menu generation (Wireless features → Connect to EOS Utility) →
instructions updated for both generations. Shutter silent → tracer dumped
**0xc1b6**, an object-added record in NO documentation; decoded from hex
(oid@0 invariant held) → 3-line fix. Camera was shooting 40 MB CR3 RAW →
thumb sidecars for the grid + JPEG advisory. **Full pipeline proof #2:**
`captures/r6-test/2026-07-08T22-53-23/` with 4 CR3s.

## Mistakes worth remembering
- Trusting documentation over instrumentation (event layouts) — twice.
- Fixing without a visible version indicator (stale-cache round).
- Two plausible culprits at once (Spotify AND TCC) — kill one completely
  before moving on.
